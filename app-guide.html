8UP Guide

앱 설명

8UP은 회원이 사용하는 사용자 앱과 스튜디오 운영자가 사용하는 관리자 웹으로 구성됩니다. 두 화면은 하나의 스튜디오 운영 데이터를 함께 사용하며, 예약, 수강권, 공지, 취소 정책이 서로 연결되어 동작합니다. 이 문서는 그 연결 구조를 실제 화면 순서대로 정리한 안내 문서입니다.

사용자 앱 + 관리자 웹 통합 안내 최종 수정일: 2026-03-19

서비스 개요

기존 안내 문서에서 다루던 핵심 역할과 연결 구조를 먼저 정리했습니다.

사용자 앱

회원 예약, 취소 요청, 수강권 확인, 공지와 이벤트 확인

관리자 웹

수업 개설, 회원 관리, 수강권 발급, 운영 정책 설정

핵심 연결 구조

수업 템플릿 → 수강권 연결 → 수업 개설 → 회원 예약

운영 기준

스튜디오에서 설정한 정책과 노출 기준이 앱에 그대로 반영됩니다.

운영 설명

스크린샷을 보기 전에 운영 구조와 흐름을 텍스트로 먼저 이해할 수 있게 구성했습니다.

Guide 01

8UP 전체 구조

관리자 웹에서 스튜디오 운영자가 수업 템플릿, 수업 일정, 수강권 상품, 공지, 취소 정책을 설정하면 그 정보가 사용자 앱에 반영됩니다. 회원은 사용자 앱에서 자신에게 연결된 스튜디오와 수강권 기준으로 예약 가능한 수업을 보고 예약하거나 취소 요청을 진행합니다.

Guide 02

사용자 앱에서 할 수 있는 일

사용자 앱은 회원이 실제로 수업을 이용하는 화면입니다. 회원은 스튜디오 선택, 수업 예약, 내 예약 조회, 수강권 확인, 공지 및 이벤트 확인, 알림 확인, 프로필 수정 등을 할 수 있습니다.

  • 스튜디오: 현재 연결된 스튜디오 정보와 공지, 이벤트 확인
  • 예약: 날짜별 수업 조회 후 수강권을 선택해 예약
  • 내 예약: 예정, 대기, 완료, 취소 상태를 한 번에 확인
  • 마이: 내 정보, 스튜디오 선택, 내 수강권 및 사용 이력 확인

Guide 03

사용자 앱 예약 흐름

회원은 자신이 발급받은 수강권으로 예약 가능한 수업만 볼 수 있습니다. 예약 버튼을 누르면 사용할 수강권을 선택하고 예약을 완료합니다. 스튜디오가 취소 정책을 설정한 경우, 앱은 취소 가능 여부에 따라 `취소 요청` 또는 `직접 문의` 같은 문구로 안내합니다.

대기 상태는 자동 예약으로 바뀌지 않습니다. 대기 승인 여부는 관리자 웹에서 운영자가 직접 확인하고 처리합니다.

Guide 04

관리자 웹에서 할 수 있는 일

관리자 웹은 스튜디오 운영 전용 화면입니다. 스튜디오 운영자는 회원과 수업을 관리하고, 수강권 발급과 운영 정책을 설정하며, 콘텐츠를 등록할 수 있습니다.

  • 대시보드: 매출, 환불, 월간 운영 현황 확인
  • 수업 템플릿: 반복적으로 사용할 수업 기본 정보 관리
  • 수업 관리: 달력 기준 수업 개설, 강사 지정, 정원 조정, 예약/대기 처리
  • 수강권 상품: 어떤 수업을 예약할 수 있는지 연결된 상품 관리
  • 회원 관리: 회원 검색, 수강권 발급, 이력 수정, 상담 기록 관리
  • 취소 관리: 취소 가능 시간과 앱 내 취소 요청 허용 여부 설정
  • 콘텐츠 관리: 공지와 이벤트 등록, 중요 공지 푸시 발송

Guide 05

관리자 웹 운영 흐름

일반적인 운영 순서는 먼저 수업 템플릿을 만들고, 그 템플릿을 예약 가능한 수강권 상품에 연결한 뒤, 수업 관리에서 실제 날짜 기준 수업을 개설하는 방식입니다. 회원에게 수강권이 발급되면 그 회원의 앱에 예약 가능한 수업이 보이기 시작합니다.

대기 회원이 생기면 수업 관리 화면의 `대기` 버튼에서 순번을 확인하고, 운영자가 직접 연락 후 `예약으로 변경` 또는 `대기 취소` 처리를 합니다.

Guide 06

사용자 앱과 관리자 웹이 연결되는 방식

사용자 앱에 보이는 대부분의 정보는 관리자 웹에서 입력한 값입니다. 예를 들어 공지와 이벤트는 콘텐츠 관리에서 등록한 내용이 앱에 노출되고, 수업 목록은 수업 관리에서 개설한 일정이 표시됩니다. 회원의 수강권 잔여 횟수, 예정/완료 상태, 취소 요청 가능 여부도 모두 관리자가 만든 수강권과 정책을 기준으로 계산됩니다.

Guide 07

운영자가 알아두면 좋은 점

  • 수업 템플릿만 만들면 앱에 바로 보이지 않고, 실제 수업을 개설해야 회원이 예약할 수 있습니다.
  • 수강권에 연결되지 않은 수업 템플릿은 회원 앱에서 예약 대상으로 보이지 않습니다.
  • 중요 공지 또는 중요 이벤트는 푸시 알림으로도 발송될 수 있습니다.
  • 강사 정산과 수업 집계는 수업 관리 달력에서 실제 지정된 강사 기준으로 반영됩니다.
  • 회원 탈퇴, 수강권 조정, 취소 정책 변경 전에는 실제 운영 영향 범위를 함께 확인하는 것이 좋습니다.
